# Codex 账号池需求

## 目标

在现有 macOS SwiftUI 工具中增加本地 Codex 账号池，参考 [qxcnm/Codex-Manager](https://github.com/qxcnm/Codex-Manager)，支持多账号登录、额度查看、账号预热和当前账号直连切换。

## 功能范围

### 1. 账号登录

- 通过 OpenAI OAuth 授权页登录 ChatGPT/Codex 账号。
- 使用 PKCE 和随机 `state`，登录回调只接受本次发起的会话。
- 登录成功后记录账号显示名、账号 ID、套餐信息和 token 过期时间。
- access token、refresh token、id token 只能保存到 macOS Keychain，不进入 Git、日志或普通 JSON。
- 支持从当前 `~/.codex/auth.json` 导入当前账号，作为登录失败或已有账号的兼容入口。

### 2. 账号列表和额度

每条账号显示：

- 邮箱或可识别账号名；
- 当前状态、套餐；
- 5 小时主窗口剩余百分比、预计刷新时间；
- 7 天次窗口剩余百分比、预计刷新时间；
- 最近刷新时间和最近错误。

支持刷新单个账号和刷新全部账号。接口不可用时保留最近快照，并在列表标出过期/失败，而不是清空已有额度。

### 3. 账号预热

- 可预热选中账号；未选中时预热全部账号。
- 对每个账号使用其 token 向 Codex Responses 上游发送最小合法请求。
- 预热成功后立即重新查询该账号额度，观察上游是否产生新的 5 小时窗口。
- 预热结果逐账号显示成功或失败，不因单个账号失败中断其他账号。
- 预热不伪造或修改额度时间；5 小时刷新时间以 OpenAI 返回的 usage 数据为准。

### 4. 账号直连切换

- 在账号列表中将任一账号设为当前账号。
- 切换前备份现有 `~/.codex/auth.json`。
- 将目标账号的最新 access/refresh/id token 写回 `~/.codex/auth.json`，保持 Codex 认可的字段结构。
- 切换后刷新当前账号显示，并提示重启或重新连接 Codex 会话以让已运行进程读取新凭据。
- 切换失败时恢复备份，避免破坏当前登录状态。

## 非目标

- 第一版不实现 Codex 请求代理、自动轮换、按请求路由和远程账号服务。
- 不绕过登录、风控、地区限制或服务端额度规则。
- 不在本地计算或伪造 5 小时/7 天额度。

## 验收标准

1. 能新增至少两个账号，并在应用重启后保留账号名和额度快照。
2. 能安全刷新单个/全部账号的 5 小时与 7 天额度。
3. 能对多个账号执行预热，逐条返回结果，成功后刷新额度。
4. 能切换当前账号，切换前生成备份，目标 token 写入 `~/.codex/auth.json`。
5. 无 token 出现在普通文件、UI 日志、错误提示和 Git diff 中。
6. `swift build`、原生后端自检和应用启动验证通过。
